# Securing Systems

## 一、课程概述
- **课程主题**：聚焦网络与系统安全，核心覆盖无线网络防护、网络协议安全、端口管理、恶意软件防御等关键场景，解决“数据传输”和“系统本身”的安全问题。
- **主讲人**：David J. Malan（哈佛大学）
- **核心目标**：理解网络协议（HTTP/HTTPS/SSH）、VPN、防火墙等技术的安全逻辑，识别系统面临的网络攻击与恶意软件威胁，掌握实用防御手段。

## 二、无线网络安全：Wi-Fi
### 1. 核心安全机制
- Wi-Fi 安全的核心是**加密保护**，通过协议实现设备与网络间的数据加密传输。
- 主流加密标准：WPA（Wi-Fi Protected Access），用于保障 secured 网络的数据安全；unsecured 网络无加密，数据易被窃取。

### 2. 风险提示
- 公共 Wi-Fi（无加密或弱加密）是高风险场景，易遭受中间人攻击、数据包嗅探等威胁，避免在公共 Wi-Fi 中传输敏感信息（如银行卡号、密码）。

## 三、网络协议安全：HTTP 与 HTTPS
### 1. HTTP（超文本传输协议）
- **本质**：未加密的明文传输协议，数据在网络中裸奔。
- **核心风险**：
  - 中间人攻击：黑客注入恶意 HTML 代码（如广告、恶意脚本）。
  - 数据包嗅探：窃取传输中的敏感数据（如信用卡号、登录凭证）。
  - 会话劫持：通过恶意 Cookie（如超级 Cookie）跟踪用户或劫持登录会话。

### 2. HTTPS（安全超文本传输协议）
- **本质**：HTTP 的加密版本，基于 TLS（传输层安全协议）和公钥加密实现数据安全传输。
- **安全原理**：
  1. 网站持有“公钥+私钥”对，公钥通过 X.509 证书发布，证书由可信第三方 CA（证书颁发机构）签名。
  2. 浏览器访问网站时，下载并验证证书：用 CA 公钥解密证书签名，与证书本身的哈希值比对，一致则确认网站身份合法。
  3. 双方通过公钥加密协商共享密钥，后续数据用对称加密传输（兼顾安全与效率）。
- **防御攻击**：
  - 针对“SSL 剥离攻击”（黑客将 HTTPS 流量重定向到恶意 HTTP 网站），可启用 HSTS（HTTP 严格传输安全）：服务器强制浏览器仅使用 HTTPS 连接。

## 四、虚拟专用网络：VPN
- **定义**：在公共网络中建立加密通信通道，实现两个节点间的安全数据传输。
- **核心特性**：
  - 全流量加密：通道内所有数据均被加密，防止中间人窃取。
  - IP 伪装：用户设备将使用 VPN 服务器的 IP 地址，隐藏真实 IP（常被用于跨区域访问服务）。
- **注意事项**：选择可信的 VPN 服务商，避免使用未知来源的 VPN（可能泄露数据）。

## 五、远程安全访问：SSH
- **定义**：安全外壳协议（Secure Shell），用于通过加密方式远程登录服务器并执行命令。
- **核心价值**：替代未加密的 Telnet 协议，防止远程操作过程中命令、凭证被窃取。
- **使用示例**：`ssh stanford.edu`（需提供合法凭证和权限，即可远程操作斯坦福大学服务器）。
- **默认端口**：22（端口号与服务绑定，是防御重点）。

## 六、端口与系统防护
### 1. 端口的核心作用
- 端口是服务器上“服务的入口”，用数字标识（0-65535），不同端口对应不同服务：
  - 80 端口：HTTP 服务。
  - 443 端口：HTTPS 服务。
  - 22 端口：SSH 服务。
- 服务器通过“监听端口”接收外部请求，端口暴露是系统主要攻击入口之一。

### 2. 端口相关风险与防御
- **端口扫描**：黑客尝试连接所有端口，识别开放端口并寻找漏洞。
- **防御手段**：
  - 防火墙（Firewall）：软件或硬件层面阻断未授权端口访问，基于 IP 地址、端口号过滤流量。
  - 深度包检测（Deep Packet Inspection）：通过代理设备分析数据包内容，拦截恶意数据（如公司监控敏感信息外发）。
  - 渗透测试/伦理黑客：专业人员主动扫描端口、寻找漏洞，提前修复安全隐患（合法合规的安全测试）。

## 七、恶意软件（Malware）与防御
### 1. 常见恶意软件类型
| 类型 | 定义与危害 |
|------|------------|
| 病毒（Virus） | 依附于其他软件安装，可篡改系统、窃取数据，需通过文件传播（如邮件附件、恶意软件安装包） |
| 蠕虫（Worm） | 无需依附其他软件，通过系统漏洞自主传播（如网络漏洞、U盘），可快速感染大量设备 |
| 僵尸网络（Botnet） | 感染设备后将其纳入控制网络，黑客可远程下达指令，发起分布式拒绝服务（DDoS）攻击（通过海量设备发送请求，瘫痪目标服务器） |

### 2. 杀毒软件（Antivirus）
- **核心功能**：检测并清除病毒、蠕虫等恶意软件，基于病毒库匹配、行为分析识别威胁。
- **使用关键**：
  - 启用自动更新：及时更新病毒库和软件本身，修复旧版本安全漏洞。
  - 局限性：无法防御“零日攻击”（利用未知软件漏洞的攻击，厂商尚未发布补丁）。

## 八、关键总结与注意事项
1. 网络安全的核心是“加密+认证”：HTTPS、VPN、SSH 均通过加密保障传输安全，证书、凭证确保通信双方身份合法。
2. 端口是系统“暴露的窗口”：关闭不必要的开放端口，用防火墙严格管控端口访问。
3. 恶意软件防御需“技术+意识”：依赖杀毒软件的同时，避免下载未知来源的软件、打开可疑邮件附件。
4. 公共网络风险高：在公共 Wi-Fi 中不传输敏感信息，必要时使用 VPN 加密。
5. 技术防护有局限：HSTS 防御 SSL 剥离，但无法防范 CA 证书伪造；杀毒软件无法应对零日攻击，需定期更新系统和软件。
