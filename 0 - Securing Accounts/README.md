# Securing Accounts

## 一、课程概述
- **课程主题**：聚焦账户安全核心，涵盖安全基础、常见攻击手段及防御策略，核心是平衡“安全性”与“可用性”。
- **主讲人**：David J. Malan（哈佛大学）
- **核心目标**：理解数字安全的核心逻辑，识别常见账户攻击方式，掌握保护登录凭证的实用方法。

## 二、安全基础核心概念
### 1. 安全的本质与核心要素
- 数字世界的安全类比：类似现实世界的“钥匙-锁”，核心是**授权验证**——确认用户是账户的合法所有者。
- 授权的两大基础：
  - 用户名：用户身份的“声明”（表明“你是谁”）。
  - 密码：用户身份的“证明”（验证“你确实是你”）。
- 核心权衡：安全性与可用性成反比，高度安全的方案可能降低使用便捷性，需根据场景选择适配方案。

### 2. 常见密码攻击方式
- 字典攻击：黑客使用预设的字典词库（如常见密码、英文单词）猜测密码。
- 暴力破解：遍历所有可能的字符组合，尝试破解密码（效率取决于密码复杂度）。

## 三、密码复杂度与防御效果
### 1. 密码组合可能性对比
密码的安全强度直接取决于“字符集大小”和“长度”，不同组合的破解难度差异显著：

| 密码规则 | 字符集范围 | 组合数量 | 破解难度 |
|----------|------------|----------|----------|
| 4位纯数字 | 10个数字（0-9） | 10×10×10×10=10,000 | 极低（毫秒级破解） |
| 4位大小写字母 | 52个字母（a-z、A-Z） | 52×52×52×52≈700万 | 较低（短时间破解） |
| 4位字母+数字+标点 | 78个字符（52字母+10数字+16标点） | 78×78×78×78≈3800万 | 中等（显著增加破解时间） |

### 2. 代码验证（Python示例）
- 4位纯数字密码遍历：
  ```python
  from string import digits
  for i in digits:
      for j in digits:
          for k in digits:
              for l in digits:
                  print(i, j, k, l)
  ```
- 核心结论：扩大字符集、增加密码长度，能大幅提升黑客的时间成本，是防御暴力破解的关键。

## 四、NIST 安全指南（核心推荐）
美国国家标准与技术研究院（NIST）提供账户安全最佳实践，核心建议如下：
1. 记忆类密码（如登录密码）长度至少8字符。
2. 支持所有打印ASCII字符和Unicode符号，最长可至64字符。
3. 验证密码时，需排查字典词、重复序列、泄露密码列表及场景相关词（如用户名、生日）。
4. 不允许未认证用户访问密码提示。
5. 不强制定期修改密码（频繁修改易导致用户使用弱密码）。
6. 限制失败登录次数，锁定潜在攻击者。

## 五、多因素认证（2FA/MFA）
### 1. 三大认证要素
多因素认证要求用户提供两种及以上“不同类型”的凭证，大幅提升安全性：
- 知识因素（Something you know）：仅用户知晓的信息（如密码、安全问题答案）。
- 持有因素（Something you have）：仅用户拥有的设备/物品（如手机、硬件密钥）。
- 固有因素（Something you are）：用户自身的生物特征（如指纹、面部识别）。

### 2. 一次性密码（OTP）
- 定义：仅使用一次的临时密码，作为多因素认证的“持有因素”补充。
- 常见形式：
  - 短信OTP：通过短信接收，易受“SIM交换攻击”（黑客克隆SIM卡获取短信），安全性较低。
  - 认证App OTP：通过手机安全App（如Google Authenticator）生成，无需网络，安全性更高。

## 六、常见账户攻击手段与防御
### 1. 按键记录（Keylogging）
- 原理：黑客在设备上安装恶意软件，记录用户按键操作，窃取用户名、密码、OTP等信息。
- 防御：仅使用自己可控的设备登录账户，避免使用公共电脑或可疑设备。

### 2. 凭证填充（Credential Stuffing）
- 原理：利用某网站泄露的“用户名-密码”列表，尝试登录其他网站（依赖用户在多平台使用相同密码）。
- 防御：为每个网站设置唯一密码，避免密码复用。

### 3. 社会工程学（Social Engineering）
- 原理：不依赖技术，通过社会压力、信任伪装获取用户凭证（如冒充客服、熟人索要密码、套取个人信息）。
- 防御：警惕陌生人的信息询问，不轻易透露密码、生日、宠物名等敏感信息。

### 4. 钓鱼（Phishing）
- 原理：结合社会工程学与技术手段，伪装成可信网站（如谷歌、银行登录页），诱导用户输入凭证。
- 防御：不点击邮件、短信中的可疑链接，通过浏览器手动输入官方URL登录。

### 5. 中间人攻击（Machine-in-the-Middle）
- 原理：黑客劫持用户与服务端之间的中间设备（如路由器、交换机），窃取传输数据。
- 防御：优先使用HTTPS加密连接，避免在公共Wi-Fi环境下进行敏感操作（如转账、登录银行账户）。

## 七、提升账户安全的实用工具
### 1. 单点登录（SSO）
- 定义：使用谷歌、脸书等可信平台的账号，登录其他第三方服务（无需单独注册账号）。
- 优势：减少密码数量，降低复用风险，登录便捷且安全。

### 2. 密码管理器（Password Managers）
- 定义：专门管理复杂密码的软件，可自动生成、存储密码，支持多平台同步。
- 核心优势：
  - 无需记忆多个复杂密码，仅需记住一个“主密码”。
  - 能识别钓鱼网站，避免在虚假页面输入密码。
- 区别于浏览器保存：独立软件，支持跨浏览器、跨设备使用，安全性更高。
- 风险：所有密码依赖主密码，主密码泄露会导致所有账户风险。

### 3. 密钥（Passkeys）
- 定义：新兴安全技术，基于非对称加密（公钥+私钥）的自动生成密码。
- 工作原理：服务端存储公钥，用户设备存储私钥，登录时无需输入密码，通过加密验证身份。
- 优势：彻底避免密码泄露风险，未来将成为主流登录方式（需结合加密知识深入理解）。

## 八、关键总结与注意事项
1. 核心原则：账户安全的核心是“提高攻击者成本”与“降低自身风险”，平衡安全性与可用性。
2. 人为因素：用户的行为和安全意识是关键，技术工具仅为辅助（如警惕钓鱼、不复用密码）。
3. 工具选择：根据场景选择适配工具——多平台登录用SSO，复杂密码管理用密码管理器，追求极致安全可尝试Passkeys。
4. 常见误区：
  - 认为“定期改密码更安全”（NIST不推荐，易导致弱密码）。
  - 过度依赖短信OTP（忽略SIM交换攻击风险）。
  - 信任浏览器保存的密码（安全性低于专业密码管理器）。
